# 中国象棋 Web 对战

一个基于 React + Vite 的网页中国象棋项目，面向个人使用和快速迭代。项目支持人机对战、AI 自动对战、残局加载与残局推演，并内置《适情雅趣》残局库。

## 功能

- 我要下棋：玩家可选择红方或黑方，与电脑对弈。
- AI 对战：红黑双方由 AI 连续行棋。
- 残局研究：加载残局后可让 AI 双方继续推演。
- 难度设置：入门、棋士、大师、宗师四档；大师和宗师统一使用 Fairy-Stockfish WASM 引擎，宗师拥有更长搜索预算。
- 残局库：内置 471 个局面，其中包含《适情雅趣》第001-468局红胜段及 3 个精选练习局；第469局以后偏和棋/难胜类题目不进入红胜库。
- 移动端适配：手机端优先显示棋盘，并提供快捷操作条。
- 本地状态恢复：刷新页面后恢复当前局面、着法记录、设置和重开目标。
- 棋盘体验：实木质感棋盘、实体比例棋子、可走/可吃/上一步位置提示。
- 提示音：落子、吃子、将军、胜负和 UI 操作均使用开源/CC0 许可素材；吃子和将军支持中文语音播报。
- 重复局面保护：AI 自动行棋遇到重复局面会暂停，并在控制面板和分析面板提示。

```bash
npm install
npm run dev
```

```bash
npm run dev          # 启动开发服务器
npm run build        # 生产构建
npm test             # 运行 AI 与残局库校验

npm run check        # 构建并运行全部校验
npm run preview      # 预览构建结果
```

## 使用说明

- `我要下棋`：玩家对电脑。主按钮是 `电脑走一步`，用于让当前方由电脑走一步。
- `AI对战`：AI 同时控制红黑双方，适合观察不同难度对局。
- `残局研究`：加载残局后由 AI 自动延续局面，适合观察残局变化。
- `大师/宗师`：两档共用同一套 WASM 强引擎；大师搜索更快，宗师搜索更久，残局会额外加时。若浏览器暂时无法加载 WASM，分析面板会显示“本地回退”。
- 自动推演：如果 AI 走回历史局面，程序会停止自动行棋，避免车帅来回移动造成长时间卡住。
- `重开`：会先弹确认框。若当前来自残局或自定义 FEN，会回到该局面起点，而不是标准开局。
- 刷新页面：自动恢复上次局面，但不会恢复“正在自动对战”的运行状态，避免打开页面后 AI 自动走棋。

## 项目结构

```text
src/
  App.jsx                 # 顶层状态、AI 调度、本地持久化
  components/             # 棋盘、控制面板、分析面板
  game/                   # 象棋规则、AI、残局、引擎协议
  ui/                     # UI 配置
public/
  engine/                 # Fairy-Stockfish WASM 引擎文件
  sounds/                 # Kenney 与 Checkora 音效素材
scripts/
  ai-smoke.mjs            # AI 基础行为校验
  interaction-smoke.mjs   # 悔棋、编号搜索、重复局面校验
  red-win-sample.mjs      # 红胜残局抽样推演
  validate-library.mjs    # 残局库合法性校验
```

## 残局数据口径

本地保留《适情雅趣》李浭 551 局原始 FEN 作为来源数据，但应用入口只加载第001-468局红胜段。第469局开始包含“和”“难胜”等结果描述，不符合“红方先行且红方必胜”的产品目标，因此不展示、不抽样。
